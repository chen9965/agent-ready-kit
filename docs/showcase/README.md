# Real Repository Showcase / 真实仓库扫描示例

These examples were generated by running `agent-ready-kit` locally against shallow clones of well-known open-source repositories on 2026-06-30.

这些示例是在 2026-06-30 用 `agent-ready-kit` 本地扫描几个知名开源仓库得到的结果。

The goal is not to rank these projects. The goal is to show what the tool surfaces on real repositories: missing agent instructions, repo-map gaps, CI/test signals, large files, and possible secret-like files that need review.

这里不是给这些项目排名，而是展示工具在真实仓库里会发现什么：是否缺少 Agent 说明、是否缺少仓库地图、测试和 CI 信号是否明确、是否有超大文件、是否存在需要复核的疑似敏感文件。

Important note: `possible-secrets` means "review needed". It can match test fixtures, example credentials, or files whose names/content look sensitive. It is not a claim that a repository leaked real credentials.

重要说明：`possible-secrets` 的含义是“需要复核”。它可能命中测试 fixture、示例凭据，或名称/内容看起来敏感的文件；它不是在断言仓库泄漏了真实密钥。

## Command / 运行命令

```bash
node dist/cli.js scan <repo-path> --out .agent-ready/showcase-results/<repo>
```

Each run now produces four files:

每次扫描都会生成四个文件：

| File | Purpose | 中文说明 |
| --- | --- | --- |
| `scan.json` | Raw result for machines, CI, dashboards, and comparisons. | 原始扫描结果，适合机器读取、CI、仪表盘和对比。 |
| `report.md` | Human-readable score, breakdown, and findings. | 人能直接看的分数、评分明细和问题。 |
| `before-after.md` | Explains the difference between guessing the repo and having scan artifacts. | 解释没有工具时只能猜什么、扫描后多了哪些明确产物。 |
| `action-plan.md` | Prioritized next steps for improving agent readiness. | 提升 Agent 就绪度的优先级行动计划。 |

## What This Shows / 这说明了什么

- Big mature repositories usually score well on README, tests, and CI.
- Agent-specific onboarding is still uneven: some large projects have `AGENTS.md`, some do not.
- Repo maps matter even in famous projects, because agents need local orientation, not brand familiarity.
- Safety findings are intentionally conservative and should be reviewed by humans.

- 成熟大仓库通常在 README、测试、CI 上表现不错。
- Agent 专属上手体验仍然不均衡：有些大项目已经有 `AGENTS.md`，有些没有。
- 仓库地图很重要，因为 Agent 需要的是本地结构说明，不是“这个项目很有名”。
- 安全类发现故意偏保守，应该由人复核。
